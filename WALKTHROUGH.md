# 攻略マニュアル（ネタバレ注意！）

> このドキュメントはARG運営者向けの正解マニュアルです。
> プレイヤーには見せないでください。

---

## ゴール

「SEIKAI CAREER SCOUT」という一見普通の転職サイトを、
「ISEKAI CAREER SCOUT」という異世界キャリアサイトに変身させること。

---

## 攻略手順（スクリーンショット付きガイド）

### ステップ 1：サイトを開く

普通の転職スカウトサイトが表示されます。
一見何も怪しくないサイトですが、ソースコードにヒントが隠されています。

---

### ステップ 2：ヒントを見つける

ヒントは2箇所にあります。どちらか片方でも見つければOKです。

#### ヒント① ソースコードを見る（簡単）

1. サイト上で **右クリック**
2. **「ページのソースを表示」** をクリック（Chromeの場合）
   - Mac: `Command + Option + U` でもOK
   - Windows: `Ctrl + U` でもOK
3. ソースコードが新しいタブで開きます
4. コード内にこのようなコメントが見つかります：

```
HINT: このサイトは文字化けしている...
「正解」のスペルを確認してみて。
本当にそれは「正解」なのか？
もしかしたら、別の世界(SEKAI)が見えるかもしれない。
data-key を探せ。
```

また、以下のコメントも見つかります：

```html
<!-- THE KEY: この要素の data-key を書き換えよ
     "SEIKAI" → "ISEKAI" に変更すると... -->
<div id="site-core" data-key="SEIKAI"></div>
```

#### ヒント② コンソールを見る（やや上級）

1. サイト上で **右クリック** → **「検証」**（または「要素を調査」）
2. 上部のタブから **「Console」**（コンソール）を選ぶ
3. 黄色い警告メッセージが表示されています：

```
[WARNING] charset anomaly detected in #site-core — data-key mismatch
```

→ `#site-core` という要素の `data-key` がおかしい、というヒントです。

---

### ステップ 3：コードを書き換える（ここがメインのギミック！）

1. サイト上で **右クリック** → **「検証」**（Inspect / 要素を調査）
   - Mac: `Command + Option + I`
   - Windows: `Ctrl + Shift + I` / `F12`

2. 開発者ツールが開きます。**「Elements」**（要素）タブが選ばれているか確認。

3. **`SEIKAI` を検索します。**
   - 開発者ツールの **Elementsパネル内をクリック** してから、`Command + F`（Mac）/ `Ctrl + F`（Windows）を押す
   - 検索バーが下部に出るので **`SEIKAI`** と入力してEnter
   - 下の画像のように `data-key="SEIKAI"` がハイライトされます

   > もし検索バーが出ない場合は、Elementsパネルの白い部分（HTMLのコードが並んでいるところ）を一度クリックしてからもう一度 `Cmd+F` を試してください。

4. ハイライトされた部分を見ると、こういうコードがあります：

   ```html
   <div id="site-core" data-key="SEIKAI"></div>
   ```

5. **`SEIKAI` の文字をダブルクリック** すると、文字が編集できる状態になります（青くハイライトされます）

6. **`SEIKAI` を全部消して `ISEKAI` と入力します**
   - SEIKAI → ISEKAI（先頭の「S」を「I」に変えて「I」を先頭に追加するイメージ）
   - ダブルクォーテーション（`"`）は消さないでください
   - 正しい形: `data-key="ISEKAI"`

7. **Enterキーを押す**

---

### ステップ 4：変身！

Enterを押した瞬間、以下のことが起こります：

1. 画面が紫色にフラッシュする（グリッチエフェクト）
2. サイト全体のカラーが **青（企業風）→ 紫（ファンタジー風）** に変化
3. キラキラしたパーティクル（光の粒）が浮遊し始める
4. サイトの内容が全て **異世界版** に切り替わる
5. 画面下部に「異世界への扉が開かれた...」というメッセージが表示

## Q&A

### Q: 「検証」が見つからない
- Chrome: 右クリック → 「検証」
- Safari: まず「設定」→「詳細」→「Webデベロッパ用の機能を表示」にチェック。その後右クリック → 「要素の詳細を表示」
- Firefox: 右クリック → 「要素を調査」
- Edge: 右クリック → 「開発者ツールで調査する」

### Q: data-key が見つからない
- 開発者ツールの Elements パネル内で `Ctrl+F` / `Cmd+F` で「SEIKAI」を検索

### Q: 書き換えてもなにも起きない
- `ISEKAI` が全て大文字になっているか確認（`isekai` や `Isekai` ではNG）
- ダブルクォーテーション（`"`）は消さずに、中の文字だけ変える
- 正しい形: `data-key="ISEKAI"`

### Q: 元に戻したい
- ページをリロード（`F5` / `Cmd+R`）すれば元のサイトAに戻ります

## 技術メモ（運営者向け）

- 仕組み: `MutationObserver` が `#site-core` の `data-key` 属性の変更を監視
- `SEIKAI` → `ISEKAI` に変わった瞬間 `transformToIsekai()` が発火
- 全てクライアントサイドで完結（サーバー不要）
- GitHub Pages で静的ホスティング可能
